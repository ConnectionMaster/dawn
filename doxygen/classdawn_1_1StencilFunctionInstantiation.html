Detailed Description

Specific instantiation of a stencil function.

Fields of a stencil function instantiation are mappend to the fields of the caller i.e all fields are mapped to one of the fields of the "main" stencil. Arguments are identified by their index in the argument list. For each argument of the function we record the following quantities, depending on their type:

  • Argument is a direction: We record a mapping of the arugment index to the dimension the direction maps to
    See also
    getCallerDimensionOfArgDirection().
  • Argument is an offset: We record a mapping of the argument index to the dimension and dimensional offset
    See also
    getCallerOffsetOfArgOffset().
  • Argument is a field: We record a maping of the argument index to the AccessID the field maps to
    See also
    getCallerAccessIDOfArgField().
    In addition, we store the initial offset of each field as specified in the argument list (e.g calling avg(u(i+1)) will yield an initial offset [1, 0, 0] for each access to the field u maps to).

Example: Consider the following example (given in the gridtools_clang DSL):

stencil_function avg {
storage in;
direction dir;
Do { return in(dir+1); }
};

Now, we call avg with field u (which we assume u has AccessID 5):

out = avg(u(i+1), i);

The members of the StencilFunctionInstantiation for this call:

Member Values Explanation
ArgumentIndexToCallerAccessIDMap {0, 5} Field in maps to field u
ArgumentIndexToCallerDirectionMap {1, 0} Direction dir maps to dimension 0
ArgumentIndexToCallerOffsetMap {} No offsets
CallerAcceessIDToInitialOffsetMap {5, [1, 0, 0]} avg is called with u at i+1
