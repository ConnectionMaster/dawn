Extents::VerticalLoopOrderAccess dawn::Extents::getVerticalLoopOrderAccesses ( LoopOrderKind  loopOrder) const

Check if there is a stencil extent (i.e non-pointwise) in the counter-loop- and loop order.

If the loop order is forward, positive extents (e.g k+1) in the vertical are treated as counter-loop order accesses while negative extents (e.g k-1) are considered loop order accesses and vice versa for backward loop order. If the loop order is parallel, any non-pointwise extent is considered a counter-loop- and loop order access.
